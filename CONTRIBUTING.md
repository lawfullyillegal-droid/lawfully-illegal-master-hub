#### JavaScript/Node.js
- Use ES6+ features
- Follow consistent naming conventions
- Add JSDoc comments for functions
- Include error handling
- Write modular, reusable code

#### File Structure
```
src/
  api/          # API route handlers
  data/         # Legal definitions and data
  config/       # Configuration files
scripts/        # Automation and deployment scripts
```

#### Naming Conventions
- Files: `kebab-case.js`
- Functions: `camelCase()`
- Constants: `UPPER_SNAKE_CASE`
- Classes: `PascalCase`

### 4. Pull Request Process

1. **Fork the repository**
   ```bash
   git clone https://github.com/lawfullyillegal-droid/lawfully-illegal-master-hub.git
   cd lawfully-illegal-master-hub
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow coding standards
   - Add/update documentation
   - Include tests if applicable

4. **Verify legal accuracy**
   - Double-check all statutory citations
   - Verify constitutional references
   - Ensure definitions are accurate

5. **Test your changes**
   ```bash
   npm install
   npm test
   npm run dev  # Test locally
   ```

6. **Commit with clear messages**
   ```bash
   git add .
   git commit -m "Add: UCC Article 4A definitions for electronic funds transfer"
   ```

7. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

8. **Submit a Pull Request**
   - Use a clear, descriptive title
   - Reference related issues
   - Describe what changed and why
   - Include legal citations for content changes
   - List any breaking changes

### 6. Issue Reporting

When reporting issues:

#### Bug Reports
- Describe the bug clearly
- Include steps to reproduce
- Provide error messages
- List your environment (OS, Node version)

#### Legal Accuracy Issues
- Cite the incorrect definition/citation
- Provide correct citation with source
- Include authoritative reference
- Explain the legal significance

#### Feature Requests
- Describe the feature clearly
- Explain the use case
- Suggest implementation approach
- Note any legal basis if applicable
